## 3) Decisions Locked
- Product/technical decisions made this phase:
  - The benchmark focus overlay stays compact and anchored to the viewer top-right instead of the previous lower-right presentation.
  - OCC-dependent CLI tools should be interpreter-aware, not shell-aware. They must detect whether the current Python can import `FreeCAD` and `OCC`, and re-launch under a compatible FreeCAD Python automatically when needed.
  - Dated benchmark manifests must resolve repo-root from the manifest's discovered repo root, not from simple parent-folder assumptions.
  - The viewer left panel should preview mapped standards as a short chip list with an explicit `+N more` summary and optional full expansion, instead of dumping the full standards string inline.
  - Browser regression coverage for the compact overlay should mock import/API contracts and assert viewer-relative placement, so the UI check stays fast and deterministic.
- Assumptions applied:
  - The correct FreeCAD Python on this machine is `C:\Program Files\FreeCAD 1.0\bin\python.exe`.
  - Plain shell `python` remains Anaconda on this machine, so benchmark tools must not rely on shell interpreter alignment.
  - A dedicated Playwright dev server can run on `http://127.0.0.1:45173` without sharing state with an already-running local Vite session.
- Defaults chosen:
  - Benchmark runner still prefers `plans/20260313/dfm_mtk_benchmark_manifest.local.json`.
  - Feature parity reporter still prefers the same dated manifest path when present.
  - Browser regression coverage runs through `npm run test:e2e` before the web build in CI.
